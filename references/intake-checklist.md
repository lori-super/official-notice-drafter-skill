# Intake Checklist

Use this checklist before drafting.

## Required fields

1. Topic and objective
- What problem does this notice solve?
- Is there any upstream requirement/policy to cite?

2. Time information
- Start time, end time, and deadline
- Any milestone times

3. Scope and audience
- Recipients
- Affected teams/sites/systems/people

4. Execution requirements
- What actions are required
- Any hard constraints (must/forbidden/prerequisites)

5. Signature and date
- Signature default is already configured in SKILL.md
- Date default is today

## Optional fields

- Contacts (default: omit)
- Attachments
- CC list

## Follow-up template

Use this when fields are missing:

"To ensure this notice can be issued directly, please add: 1) ... 2) ... 3) ..."
