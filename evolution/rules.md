# 生效规则（仅由“定稿可发”样本更新）

## 0. 采纳闸门（强约束）
- 仅接受两类判定：
  - `定稿可发`
  - `不采用：原因标签`
- 只有 `定稿可发` 才允许更新本文件。
- `不采用` 只记录到 rejected，不修改主规则。

## 1. 结构规则
- 默认输出：标题 + 称谓 + 正文（可按用户要求一段/两段）+ 落款 + 日期。
- 正文必须包含：背景原因、风险/影响、具体需求明细、执行请求。

## 2. 语气规则
- 使用正式公文语气，避免口语化。
- 用词要求明确可执行，避免模糊词（如“尽快”“适当”）。

## 3. 必含信息
- 重点场景与风险说明（为什么要做）。
- 布线/资源明细（来源点、目标点、数量、用途）。
- 关键保障诉求（冗余、连续性、安全性）。

## 4. 禁用项（从 rejected 累积）
- [待补充：如“语气过软”]
- [待补充：如“只写需求不写风险”]

## 5. 追问触发条件
- 若缺少时间安排、收文对象、落款、数量口径冲突，先追问再成文。
