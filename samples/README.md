# 样本使用说明（每次出稿后立即迭代）

本目录用于沉淀通知写作样本，并通过“采纳闸门”持续优化 skill。

## 1) 判定口令（只用这两种）
- `定稿可发`
- `不采用：原因标签`

> 只有“定稿可发”才允许更新规则；不采用样本仅进入反例库。

## 2) 目录说明
- `accepted/`：定稿样本（正样本）
- `rejected/`：不采用样本（反例）

## 3) 单次迭代流程（固定 5 步）
1. 保存本次需求与输出。
2. 给出判定口令。
3. 若“不采用”：仅记录标签与点评，不改主规则。
4. 若“定稿可发”：提炼 1-3 条规则更新 `evolution/rules.md`。
5. 用 3 条历史案例做回归，并记录到 `evolution/changelog.md`。

## 4) 防过拟合规则
- 同一负面标签连续出现 >= 2 次，才进入主规则评估。
- 单次个性化偏好，不直接写入主规则。

## 5) 标签建议（统一口径）
- 语气不够正式
- 逻辑顺序不对
- 重点风险没写透
- 细节数量不准
- 结尾执行力不足
- 术语不符合习惯

## 6) 模板文件
- 正样本模板：`accepted/CASE_TEMPLATE.md`
- 反例模板：`rejected/CASE_TEMPLATE.md`
